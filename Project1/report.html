Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 7 - Busca em Matriz¶

  • O que a tcnica de busca em matriz (grid search)?
  • Como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Dica: Quando explicar a tcnica de busca em matriz, tenha certeza que voc explicou o motivo dela ser usada, o que a 'matriz' significa nesse caso e qual o objetivo da tcnica. Para ter uma resposta mais slida, voc pode tambm dar exemplo de um parmetro em um modelo que pode ser otimizado usando essa tcnica.

Resposta: A tcnica de busca em matriz uma mtodo para escolher os melhores hiperparmetros do modelo. Onde a matriz representa todas as possibilidades de combinaes de hiperparmetros que temos. Dessa forma, ns podemos usar tal matriz para iterar sobre tais combinaes, avaliar cada uma delas, e escolher a configurao que construiu o melhor modelo. Por exemplo: Digamos que queremos otimizar os parmetros Kernel e Gamma de uma SVM de modo a obter o modelo com melhor acurcia, ns ento podemos construir a seguinte matriz:

Gamma\kernel rbf linear
1 acurcia= 55% acurcia=75%
10 acurcia= 65% acurcia=70%
100 acurcia1= 80% acurcia=68%

Para cada combinao de Kernel e Gamma, ns criamos um modelo, avaliamos o mesmo considerando a acurcia como mtrica, e por fim, aplicamos o melhor modelo nos dados de teste, a fim de garantir que nosso modelo tem uma boa capacidade de generalizao. recomendvel aplicar uma tcnica de crossvalidation quando decidir avaliar o modelo, a fim de ter uma avalio mais confivel da performance do mesmo. Por fim, ns usamos a melhor combinao para treinar um modelo final com todos os nossos dados.

Resposta: Validao-cruzada k-fold uma tcnica para estimar a performance do modelo. Tal tcnica consiste em dividir nosso dados de treino em "K" partes, e para cada parte ns aplicamos o algoritmo abaixo:

- Para cada "K" parte faa:
    - Usamos "k-1" partes para treinar um modelo
    - Avaliamos o modelo treinado com a parte que restou

A mdia das mtricas de performances calculadas em cada iterao , ento, retornada como resposta. Essa tcnica pode ser facilmente usada junto com a busca em matriz na escolha da melhor configurao de hiperparmetros. Sem ela, existe uma chance maior da escolha dos hiperparmetros sofrer overfitting, pois os mesmos seriam calibrados usando apenas os dados fixos de treino e teste. Uma soluo parcial seria dividir os dados em trs partes: Treino, Validao e Teste. Dessa forma, usaramos o conjunto de validao para avaliar o modelo em cada iterao, e no fim do processo, usaramos o conjunto de Teste para uma avaliao final. Contudo, tal mtodo diminui mais os dados de treino e ficamos dependentes de uma escolha randomica dos conjunto de treino e validao. Tais caracteristicas no se apresentam no mtodo de K-fold, pois o mesmo no necessita de um conjunto validao fixo e, tambm, garante que todos os dados de treino so usados tanto para treinar quanto pra validar uma vez cada.

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados

    # sklearn verso 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
