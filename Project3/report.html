Implementação: Detecção de valores atípicos (Outlier)¶

Identificar dados discrepantes extremamente importante no passo de pr-processamento de dados de qualquer anlise. A presena de discrepantes podem enviesar resultados que levam em considerao os pontos de dados. H muitas "regras bsicas" que constituem um discrepante em um conjunto de dados. Aqui usaremos o Mtodo Turco para identificar valores atpicos: Um passo do discrepante calculado 1,5 vezes a variao interquartil (IQR). Um ponto de dados com um atributo que est alm de um passo de um discrepante do IQR para aquele atributo, ele considerado anormal.

No bloco de cdigo abaixo, voc vai precisar implementar o seguinte:

  • Atribuir o valor do 25 percentil do atributo dado para o Q1. Utilizar np.percentile para isso.
  • Atribuir o valor do 75 percentil do atributo dado para o Q3. Novamente, utilizar np.percentile.
  • Atribuir o clculo de um passo do discrepante do atributo dado para o step.
  • Remover opcionalmentos os pontos de dados do conjunto de dados ao adicionar ndices lista de outliers.

NOTA: Se voc escolheu remover qualquer discrepante, tenha certeza que a amostra de dados no contm nenhum desses pontos!
Uma vez que voc executou essa implementao, o conjunto de dado ser armazenado na varivel good_data!

# Para cada atributo encontre os pontos de dados com mximos valores altos e baixos

"""
    O dicionario abaixo representa a quantidade de outliers por index. Sendo que seleciono
    apenas os indexes com pelo menos um outlier. Esse dicionario vai ser usado para remover
    os pontos que contenham pelo menos dois outliers.
"""

outlier_amount_by_index_dict = {}

for feature in log_data.keys():
    
    # TODO: Calcule Q1 (25 percentil dos dados) para o atributo dado
    Q1 = np.percentile(log_data[feature], 25)
    
    # TODO: Calcule Q3 (75 percentil dos dados) para o atributo dado
    Q3 = np.percentile(log_data[feature], 75)
    
    # TODO: Utilize a amplitude interquartil para calcular o passo do discrepante (1,5 vezes a variao interquartil)
    step = 1.5*(Q3-Q1)
    
    #Abaixo segue o codigo que atualiza o dicionario
    outlier_index_list = log_data[~((log_data[feature] >= Q1 - step) & (log_data[feature] <= Q3 + step))].index.tolist()
    
    for i in outlier_index_list:
        if i in outlier_amount_by_index_dict.keys():
            outlier_amount_by_index_dict[i] += 1
        else:
            outlier_amount_by_index_dict[i] = 1
    
    # Mostre os discrepantes
    print "Data points considered outliers for the feature '{}':".format(feature)
    
    display(log_data[~((log_data[feature] >= Q1 - step) & (log_data[feature] <= Q3 + step))])
    
# OPCIONAL: Selecione os ndices dos pontos de dados que voc deseja remover
outliers  = [k for k in outlier_amount_by_index_dict.keys() if outlier_amount_by_index_dict[k] > 1]

print "List of points that will be removed: ", outliers
# Remova os valores at, caso nenhum tenha sido especificado
good_data = log_data.drop(log_data.index[outliers]).reset_index(drop = True)

Resposta:

  • Quais so as vantagens de utilizar o algoritmo de clustering K-Means?
    • Esse algorithmo se caracteriza pelo uso de centrides como mtodo de agrupamento. No caso, ele posiciona os centrides no nosso espao de pontos, e comea a moviment-los de modo a minimizar a distncia dos pontos em relao eles. Isso conhecido como hard clustering. Dessa forma, ns sabemos que um determinado ponto X est associado com, e somente com, o cluster C1, por exemplo.
      • Vantagens:
        • Simples de entender
        • Fcil de implementar
        • Geralmente bastante rpido
  • Quais so as vantagens de utilizar o algoritmo de clustering do Modelo de Mistura Gaussiano?
    • Esse algoritmo se baseia na ideia de soft clustering, onde, diferentemente do K-means, ele presume que cada ponto pode pertencer a mais de um cluster. Em outras palavras, ele usa o conceito de probabilidade bayesiana, a fim de calcular que dado um determinado ponto X, qual as chances do mesmo ser gerado por cada cluster. Dessa forma, o cluster que obtiver a maior probabilidade com X, provavelmente, o mais adequado para aquele ponto X.
      • Vantagens:
        • Dependendo do seu objetivo, o fato do algoritmo retornar uma probabilidade de cada ponto pertencer um determinado cluster pode ser considerado uma vantagem.
        • mais flexvel do que o K-means apresentando mais parmetros do que o mesmo.
  • Concluso
    • Eu pretendo utilizar o Modelo de Mistura Gaussiana, visto que mais flexvel tanto na quantidade de parmetros que pode ser alterado, o que acredito que ajude a achar o melhor agrupamento, pois podemos tentar diferentes conjuntos de parmetros. Quanto pelo fato de usar o conceito de soft clustering, pois penso que nesse caso podemos ter clientes que se encaixem em mais de um grupo. Alm disso, o dataset pequeno, ou seja, tempo de processamento no parece ser uma varivel muito relevante nesse caso.
