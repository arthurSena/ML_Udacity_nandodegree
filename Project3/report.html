Implementação: Detecção de valores atípicos (Outlier)¶

Identificar dados discrepantes é extremamente importante no passo de pré-processamento de dados de qualquer análise. A presença de discrepantes podem enviesar resultados que levam em consideração os pontos de dados. Há muitas "regras básicas" que constituem um discrepante em um conjunto de dados. Aqui usaremos o Método Turco para identificar valores atípicos: Um passo do discrepante é calculado 1,5 vezes a variação interquartil (IQR). Um ponto de dados com um atributo que está além de um passo de um discrepante do IQR para aquele atributo, ele é considerado anormal.

No bloco de código abaixo, você vai precisar implementar o seguinte:

  • Atribuir o valor do 25º percentil do atributo dado para o Q1. Utilizar np.percentile para isso.
  • Atribuir o valor do 75º percentil do atributo dado para o Q3. Novamente, utilizar np.percentile.
  • Atribuir o cálculo de um passo do discrepante do atributo dado para o step.
  • Remover opcionalmentos os pontos de dados do conjunto de dados ao adicionar índices à lista de outliers.

NOTA: Se você escolheu remover qualquer discrepante, tenha certeza que a amostra de dados não contém nenhum desses pontos!
Uma vez que você executou essa implementação, o conjunto de dado será armazenado na variável good_data!

# Para cada atributo encontre os pontos de dados com máximos valores altos e baixos

"""
    O dicionario abaixo representa a quantidade de outliers por index. Sendo que seleciono
    apenas os indexes com pelo menos um outlier. Esse dicionario vai ser usado para remover
    os pontos que contenham pelo menos dois outliers.
"""

outlier_amount_by_index_dict = {}

for feature in log_data.keys():
    
    # TODO: Calcule Q1 (25º percentil dos dados) para o atributo dado
    Q1 = np.percentile(log_data[feature], 25)
    
    # TODO: Calcule Q3 (75º percentil dos dados) para o atributo dado
    Q3 = np.percentile(log_data[feature], 75)
    
    # TODO: Utilize a amplitude interquartil para calcular o passo do discrepante (1,5 vezes a variação interquartil)
    step = 1.5*(Q3-Q1)
    
    #Abaixo segue o codigo que atualiza o dicionario
    outlier_index_list = log_data[~((log_data[feature] >= Q1 - step) & (log_data[feature] <= Q3 + step))].index.tolist()
    
    for i in outlier_index_list:
        if i in outlier_amount_by_index_dict.keys():
            outlier_amount_by_index_dict[i] += 1
        else:
            outlier_amount_by_index_dict[i] = 1
    
    # Mostre os discrepantes
    print "Data points considered outliers for the feature '{}':".format(feature)
    
    display(log_data[~((log_data[feature] >= Q1 - step) & (log_data[feature] <= Q3 + step))])
    
# OPCIONAL: Selecione os índices dos pontos de dados que você deseja remover
outliers  = [k for k in outlier_amount_by_index_dict.keys() if outlier_amount_by_index_dict[k] > 1]

print "List of points that will be removed: ", outliers
# Remova os valores atí, caso nenhum tenha sido especificado
good_data = log_data.drop(log_data.index[outliers]).reset_index(drop = True)

Questão 6¶

  • Quais são as vantagens de utilizar o algoritmo de clustering K-Means?
  • Quais são as vantagens de utilizar o algoritmo de clustering do Modelo de Mistura Gaussiano?
  • Dadas as suas observações até agora sobre os dados de clientes da distribuidora, qual dos dois algoritmos você irá utilizar e por quê.

Dica: Pense na diferença entre os clusters mais próximos ou mais isolados.

Resposta:

  • Quais são as vantagens de utilizar o algoritmo de clustering K-Means?
    • Esse algorithmo se caracteriza pelo uso de centróides como método de agrupamento. No caso, ele posiciona os centróides no nosso espaço de pontos, e começa a movimentá-los de modo a minimizar a distância dos pontos em relação à eles. Isso é conhecido como hard clustering. Dessa forma, nós sabemos que um determinado ponto X está associado com, e somente com, o cluster C1, por exemplo.
      • Vantagens:
        • Simples de entender
        • Fácil de implementar
        • Geralmente é bastante rápido
  • Quais são as vantagens de utilizar o algoritmo de clustering do Modelo de Mistura Gaussiano?
    • Esse algoritmo se baseia na ideia de soft clustering, onde, diferentemente do K-means, ele presume que cada ponto pode pertencer a mais de um cluster. Em outras palavras, ele usa o conceito de probabilidade bayesiana, a fim de calcular que dado um determinado ponto X, qual as chances do mesmo ser gerado por cada cluster. Dessa forma, o cluster que obtiver a maior probabilidade com X, provavelmente, é o mais adequado para aquele ponto X.
      • Vantagens:
        • Dependendo do seu objetivo, o fato do algoritmo retornar uma probabilidade de cada ponto pertencer à um determinado cluster pode ser considerado uma vantagem.
        • É mais flexível do que o K-means apresentando mais parâmetros do que o mesmo.
  • Conclusão
    • Eu pretendo utilizar o Modelo de Mistura Gaussiana, visto que é mais flexível tanto na quantidade de parâmetros que pode ser alterado, o que acredito que ajude a achar o melhor agrupamento, pois podemos tentar diferentes conjuntos de parâmetros. Quanto pelo fato de usar o conceito de soft clustering, pois penso que nesse caso podemos ter clientes que se encaixem em mais de um grupo. Além disso, o dataset é pequeno, ou seja, tempo de processamento não parece ser uma variável muito relevante nesse caso.

Implementação: Criando Clusters¶

Dependendo do problema, o número de clusters que você espera que estejam nos dados podem já ser conhecidos. Quando um número de clusters não é conhecido a priori, não há garantia que um dado número de clusters melhor segmenta os dados, já que não é claro quais estruturas existem nos dados – se existem. Entretanto, podemos quantificar a "eficiência" de um clustering ao calcular o coeficiente de silhueta de cada ponto de dados. O coeficiente de silhueta para um ponto de dado mede quão similar ele é do seu cluster atribuído, de -1 (não similar) a 1 (similar). Calcular a média do coeficiente de silhueta fornece um método de pontuação simples de um dado clustering.
